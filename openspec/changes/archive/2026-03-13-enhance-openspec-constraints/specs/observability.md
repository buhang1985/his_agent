# 可观测性规范

**版本**: 1.0  
**日期**: 2026-03-11  
**状态**: 新增

---

## 新增需求

### 需求：JSON 结构化日志

所有日志必须使用 JSON 格式，便于日志收集和分析系统处理。

#### 场景：日志格式
- **当** 记录日志时
- **那么** 必须使用 JSON 格式：
```json
{
  "timestamp": "2026-03-11T14:30:00.123Z",
  "level": "INFO",
  "service": "his-agent-backend",
  "version": "1.0.0",
  "traceId": "abc123def456",
  "spanId": "span789",
  "thread": "http-nio-8080-exec-1",
  "logger": "com.hisagent.service.ConsultationService",
  "message": "Consultation created successfully",
  "userId": "u_123",
  "patientIdMasked": "p_***456",
  "duration": 125,
  "environment": "prod"
}
```

#### 场景：日志字段规范
- **当** 定义日志字段时
- **那么** 必须包含：
  - `timestamp`: ISO 8601 格式时间戳
  - `level`: 日志级别（ERROR/WARN/INFO/DEBUG）
  - `service`: 服务名称
  - `traceId`: 链路追踪 ID（全链路唯一）
  - `message`: 日志消息（不含敏感数据）
  - `duration`: 操作耗时（毫秒，如适用）

#### 场景：敏感数据过滤
- **当** 记录日志时
- **那么** 必须过滤：
  - 禁止记录患者完整身份证号
  - 禁止记录患者完整手机号
  - 禁止记录患者详细住址
  - 禁止记录 API Key 和密码
  - 使用脱敏工具自动过滤

### 需求：日志级别规范

必须正确使用日志级别，区分不同重要程度的事件。

#### 场景：ERROR 级别
- **当** 发生错误时
- **那么** 使用 ERROR：
  - 系统异常（数据库连接失败、外部服务不可用）
  - 业务异常（数据验证失败、权限不足）
  - 必须包含：错误类型、错误信息、堆栈跟踪、traceId
  - 保留时长：7 年（医疗合规要求）

#### 场景：WARN 级别
- **当** 发生警告时
- **那么** 使用 WARN：
  - 触发限流
  - 使用降级方案
  - 重试操作
  - 数据异常但可处理
  - 保留时长：2 年

#### 场景：INFO 级别
- **当** 记录业务事件时
- **那么** 使用 INFO：
  - 用户登录/登出
  - 创建问诊会话
  - 生成病历
  - 访问患者记录
  - 保留时长：7 年（HIPAA 审计要求）

#### 场景：DEBUG 级别
- **当** 记录调试信息时
- **那么** 使用 DEBUG：
  - 方法入参出参
  - SQL 执行详情
  - 外部 API 请求响应
  - 仅开发/测试环境开启
  - 保留时长：30 天

### 需求：日志保留策略

日志必须根据级别和合规要求设置保留时长。

#### 场景：日志分级存储
- **当** 存储日志时
- **那么** 必须：
```yaml
logging:
  retention:
    error: 7y      # 错误日志 7 年
    warn: 2y       # 警告日志 2 年
    info: 7y       # 业务日志 7 年（审计）
    debug: 30d     # 调试日志 30 天
    
  storage:
    hot: 7d        # 热存储 7 天（SSD）
    warm: 90d      # 温存储 90 天（HDD）
    cold: 7y       # 冷存储 7 年（归档）
```

#### 场景：日志归档
- **当** 日志归档时
- **那么** 必须：
  - 每天凌晨 2 点执行归档
  - 归档文件压缩存储（gzip）
  - 归档文件命名：`app-log-YYYY-MM-DD.tar.gz`
  - 归档文件上传到对象存储
  - 本地保留最近 7 天日志

#### 场景：日志清理
- **当** 清理过期日志时
- **那么** 必须：
  - 每天检查日志保留时长
  - 自动删除超出保留期的日志
  - 清理前备份到冷存储
  - 记录清理日志

### 需求：链路追踪

所有请求必须支持全链路追踪，便于问题定位。

#### 场景：Trace ID 生成
- **当** 收到请求时
- **那么** 必须：
  - 生成全局唯一 traceId（UUID）
  - traceId 贯穿所有服务调用
  - traceId 传递到外部服务（如支持）
  - traceId 记录到所有日志

#### 场景：Span 管理
- **当** 执行子操作时
- **那么** 必须：
  - 为每个子操作创建 span
  - span 包含：开始时间、结束时间、耗时
  - span 关联到 traceId
  - 支持嵌套 span（父子关系）

#### 场景：追踪数据导出
- **当** 导出追踪数据时
- **那么** 必须：
  - 使用 OpenTelemetry 标准格式
  - 支持导出到 Jaeger/Zipkin
  - 生产环境采样率：10%
  - 错误请求采样率：100%

### 需求：日志收集与查询

必须集中收集所有服务日志，支持高效查询。

#### 场景：日志收集
- **当** 收集日志时
- **那么** 必须：
  - 使用 Filebeat/Fluentd 收集日志文件
  - 实时传输到 Elasticsearch
  - 支持多副本保证可靠性
  - 收集失败时本地缓存

#### 场景：日志查询
- **当** 查询日志时
- **那么** 必须支持：
  - 按 traceId 查询完整链路
  - 按时间范围查询
  - 按日志级别过滤
  - 按关键字搜索
  - 按用户/患者 ID（脱敏后）查询
