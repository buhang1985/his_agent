# 测试策略规范

**版本**: 1.0  
**日期**: 2026-03-11  
**状态**: 新增

---

## 新增需求

### 需求：测试金字塔结构

所有功能必须遵循测试金字塔结构，保证覆盖率和执行效率。

#### 场景：单元测试（70%）
- **当** 编写测试用例时
- **那么** 单元测试必须占 70%：
  - 后端：JUnit 5 + Mockito
  - 前端：Vitest
  - 覆盖率要求：≥ 80%
  - 执行时间：< 10ms/个

#### 场景：集成测试（20%）
- **当** 测试模块间交互时
- **那么** 集成测试必须占 20%：
  - 使用 TestContainers 启动真实依赖
  - 测试数据库交互
  - 测试外部 API 集成
  - 执行时间：< 1s/个

#### 场景：E2E 测试（10%）
- **当** 测试完整用户流程时
- **那么** E2E 测试必须占 10%：
  - 使用 Playwright
  - 覆盖核心业务流程
  - 执行时间：< 30s/个

### 需求：集成测试规范

集成测试必须使用容器化依赖，保证测试可重复性。

#### 场景：数据库集成测试
- **当** 测试数据库交互时
- **那么** 必须：
  - 使用 TestContainers 启动 MySQL
  - 每个测试使用独立 schema
  - 测试后自动清理数据
  - 禁止使用本地数据库

#### 场景：外部服务集成测试
- **当** 测试外部服务集成时
- **那么** 必须：
  - 使用 WireMock 模拟外部 API
  - 录制真实响应作为 Mock 数据
  - 测试各种响应场景（成功、失败、超时）

### 需求：E2E 测试规范

E2E 测试必须覆盖核心业务流程，在真实浏览器环境执行。

#### 场景：智能问诊流程
- **当** 运行 E2E 测试时
- **那么** 必须覆盖：
  - 用户登录
  - 创建问诊会话
  - 语音录入（Mock）
  - 转写展示
  - 病历生成
  - 保存回 HIS

#### 场景：E2E 测试环境
- **当** 执行 E2E 测试时
- **那么** 必须：
  - 使用独立测试环境
  - 所有外部服务 Mock
  - 使用测试数据
  - 测试后自动清理

### 需求：测试数据管理

测试数据必须与生产数据隔离，使用专门生成器创建。

#### 场景：测试数据生成
- **当** 创建测试数据时
- **那么** 必须：
  - 使用 Faker 库生成假数据
  - 禁止使用真实患者数据
  - 数据包含边界值
  - 数据包含异常值

#### 场景：测试数据清理
- **当** 测试完成后
- **那么** 必须：
  - 自动删除测试数据
  - 使用事务回滚
  - 或使用 DROP + 重建

### 需求：测试报告生成

所有测试必须生成详细报告，包含成功率和覆盖率。

#### 场景：单元测试报告
- **当** 运行单元测试时
- **那么** 必须生成：
  - JUnit XML 报告
  - HTML 格式报告
  - 覆盖率报告（Jaoco/V8）
  - 失败测试详细输出

#### 场景：E2E 测试报告
- **当** 运行 E2E 测试时
- **那么** 必须生成：
  - Playwright HTML 报告
  - 失败截图
  - 执行视频（可选）
  - 性能指标
