# API 接口约束规范

**版本**: 1.0  
**日期**: 2026-03-11  
**状态**: 新增

---

## 新增需求

### 需求：统一 API 响应结构

所有 REST API 响应必须遵循统一的 JSON 结构，包含状态码、消息、数据和追踪 ID。

#### 场景：成功响应
- **当** API 请求成功处理
- **那么** 返回 HTTP 200/201，响应体包含：
```typescript
{
  "code": 200,              // 业务状态码
  "message": "success",     // 友好提示
  "data": { ... },          // 业务数据
  "timestamp": 1710144000,  // 响应时间戳
  "traceId": "abc123"       // 链路追踪 ID
}
```

#### 场景：错误响应
- **当** API 请求处理失败
- **那么** 返回对应 HTTP 状态码，响应体包含：
```typescript
{
  "code": 40001,            // 业务错误码
  "message": "参数验证失败", // 友好错误提示
  "data": null,
  "timestamp": 1710144000,
  "traceId": "abc123",
  "errors": [               // 详细错误（仅验证失败时）
    {
      "field": "patientId",
      "message": "患者 ID 不能为空"
    }
  ]
}
```

### 需求：分页查询约束

列表查询接口必须支持分页，根据数据特性选择偏移量或游标分页。

#### 场景：偏移量分页（历史数据）
- **当** 查询静态或历史数据（如病历列表）
- **那么** 使用偏移量分页：
```http
GET /api/v1/consultations?page=1&size=20&sort=createdAt,desc
```
响应包含：
```typescript
{
  "items": [...],
  "pagination": {
    "page": 1,
    "size": 20,
    "total": 100,
    "totalPages": 5,
    "hasNext": true,
    "hasPrevious": false
  }
}
```

#### 场景：游标分页（实时数据）
- **当** 查询实时数据或大数据量（如语音转写片段）
- **那么** 使用游标分页：
```http
GET /api/v1/transcripts?cursor=eyJpZCI6MTAwfQ==&size=20
```
响应包含：
```typescript
{
  "items": [...],
  "nextCursor": "eyJpZCI6MTIwfQ==",
  "hasMore": true
}
```

#### 场景：分页约束
- **当** 客户端请求分页
- **那么** 必须遵守：
  - 默认页大小：20
  - 最大页大小：100
  - 默认排序：按创建时间降序
  - 禁止无限制查询（必须指定分页参数）

### 需求：API 错误码规范

所有 API 错误必须使用统一的错误码体系，便于问题定位和前端处理。

#### 场景：错误码分类
- **当** 发生错误时
- **那么** 错误码必须遵循：
```
1xxxx - 参数验证错误
2xxxx - 认证授权错误
3xxxx - 业务逻辑错误
4xxxx - 外部服务错误
5xxxx - 系统内部错误
```

#### 场景：错误码使用
- **当** 参数验证失败
- **那么** 返回 10001-19999 范围错误码，如：
  - 10001: 必填字段缺失
  - 10002: 字段格式错误
  - 10003: 字段长度超限

### 需求：API 版本管理

所有 API 必须包含版本号，支持平滑升级和向后兼容。

#### 场景：API 版本路径
- **当** 定义新 API 端点
- **那么** 路径必须包含版本号：
```
/api/v1/consultations
/api/v2/consultations  (新版本)
```

#### 场景：版本弃用
- **当** 弃用旧版本 API
- **那么** 必须：
  - 提前 3 个月通知
  - 在响应头添加 `Deprecation: true`
  - 提供迁移文档
