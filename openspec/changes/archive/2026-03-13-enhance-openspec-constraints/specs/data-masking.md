# 数据脱敏规范

**版本**: 1.0  
**日期**: 2026-03-11  
**状态**: 新增

---

## 新增需求

### 需求：患者数据脱敏规则

所有患者敏感数据在存储、传输、展示环节必须进行脱敏处理。

#### 场景：姓名脱敏
- **当** 展示或记录患者姓名时
- **那么** 必须保留首尾字符，中间替换为 `*`：
```
张三 → 张*三
李小明 → 李*明
欧阳文华 → 欧阳*华
```

#### 场景：身份证号脱敏
- **当** 存储或展示身份证号时
- **那么** 必须保留前 6 位和后 4 位，中间替换为 8 个 `*`：
```
110101199001011234 → 110101********1234
```

#### 场景：手机号脱敏
- **当** 存储或展示手机号时
- **那么** 必须保留前 3 位和后 4 位，中间 4 位替换为 `*`：
```
13812341234 → 138****1234
```

#### 场景：地址脱敏
- **当** 记录患者地址用于日志时
- **那么** 必须脱敏到区县级：
```
北京市朝阳区建国路 100 号 → 北京市朝阳区
```

### 需求：脱敏层级控制

根据用户角色和场景，实施分级脱敏策略。

#### 场景：完全脱敏（日志、审计）
- **当** 记录操作日志、审计日志时
- **那么** 必须使用完全脱敏：
  - 姓名：张*
  - 身份证：110101********1234
  - 手机号：138****1234
  - 禁止记录完整 PII 数据

#### 场景：部分脱敏（普通医护）
- **当** 非主治医生查看病历时
- **那么** 使用部分脱敏：
  - 姓名：张*三
  - 身份证：110101********1234
  - 手机号：138****1234

#### 场景：完整显示（主治医生）
- **当** 主治医生需要完整信息时
- **那么** 必须：
  - 记录访问日志
  - 需要二次授权确认
  - 完整显示所有信息

### 需求：数据库加密存储

患者敏感字段在数据库中必须加密存储。

#### 场景：加密字段
- **当** 存储以下字段时
- **那么** 必须使用 AES-256-GCM 加密：
  - 身份证号
  - 手机号
  - 详细住址
  - 医保卡号

#### 场景：加密密钥管理
- **当** 管理加密密钥时
- **那么** 必须：
  - 密钥与数据分离存储
  - 使用 KMS 管理密钥
  - 定期轮换密钥（90 天）

### 需求：脱敏审计

所有 PII 数据访问必须记录审计日志。

#### 场景：访问日志记录
- **当** 访问患者敏感数据时
- **那么** 必须记录：
  - 访问时间
  - 访问用户 ID
  - 访问的数据类型
  - 访问目的（诊疗、科研等）
  - 脱敏级别

#### 场景：异常访问检测
- **当** 检测到异常访问模式时
- **那么** 必须触发告警：
  - 非工作时间访问
  - 批量查询患者信息
  - 跨科室访问患者数据
