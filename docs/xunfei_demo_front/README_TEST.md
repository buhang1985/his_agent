# 讯飞语音转写测试页面使用说明

## 📍 访问地址

启动前端开发服务器后访问：
```
http://localhost:5173/test/xunfei-voice
```

## 🚀 快速开始

### 1. 配置讯飞 API Key

在测试页面的配置区域填写：
- **AppID**: `8bc8b3ca` (已预填)
- **API Key**: 你的讯飞实时语音转写 API Key
- **领域**: 选择"医疗"或"通用"

### 2. 获取讯飞 API Key

1. 访问 [讯飞开放平台](https://console.xfyun.cn/)
2. 注册/登录账号
3. 进入"控制台" → "我的应用"
4. 创建应用，选择"实时语音转写"服务
5. 获取 AppID 和 API Key

### 3. 开始测试

1. 点击"**开始录音**"按钮
2. 允许浏览器访问麦克风
3. 开始说话
4. 查看实时转写结果
5. 点击"**结束录音**"停止

### 🔧 技术实现

完全参照讯飞官方 demo (`docs/xunfei_demo_front/rtasr_ws_nodejs_demo/rtasr-ws-node.js`):

```javascript
// 鉴权签名 (官方 demo 算法)
function getSigna(ts) {
  let md5 = CryptoJS.MD5(config.appid + ts).toString()
  let sha1 = CryptoJS.HmacSHA1(md5, config.apiKey)
  let base64 = CryptoJS.enc.Base64.stringify(sha1)
  return encodeURIComponent(base64)
}

// 结果解析 (官方 demo 逻辑)
if (data.cn.st.type == 0) {
  // 最终结果
  str += "【最终】识别结果："
} else {
  // 中间结果
  str += "【中间】识别结果："
}
```

## 🔍 调试技巧

### 查看调试日志

页面底部的调试日志显示所有关键事件：
- ✅ 成功事件 (绿色)
- ❌ 错误事件 (红色)
- 📤 数据发送
- ⏳ 中间结果
- 💓 心跳信号

### 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| `10800` | 连接数超限 | 等待几秒后重试 |
| `35006` | 认证失败 | 检查 API Key 是否正确 |
| `10002` | 无效参数 | 检查 AppID 配置 |

## ❓ 常见问题

### Q: 为什么点击"开始录音"没有反应？
A: 检查浏览器是否允许麦克风权限，查看浏览器地址栏是否有权限请求提示。

### Q: 转写结果不准确？
A: 
1. 确保选择正确的领域 (医疗/通用)
2. 确保环境安静，减少背景噪音
3. 使用标准普通话，语速适中

### Q: 连接失败怎么办？
A:
1. 检查 API Key 是否正确
2. 检查网络连接
3. 查看调试日志中的错误码
